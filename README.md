# Virtual-Agent-Driven-SLA-Breach-Awareness-Justification-System
A ServiceNow-based system that proactively alerts users of impending SLA breaches through a Virtual Agent and collects delay justifications to improve SLA compliance and accountability
# Project Overview

This project demonstrates the design and documentation of an SLA Governance & Accountability solution in ServiceNow.

The project focuses on:
- User Interface (UI) design
- SLA workflow understanding
- Requirement analysis
- System design
- Agile planning
- User Acceptance Testing documentation

This project does not include backend development, database implementation, or custom ServiceNow scripting. It is intended as an academic prototype demonstrating the software engineering lifecycle.

# What We Built

The project demonstrates how an SLA Governance system would function inside ServiceNow.

The prototype includes interfaces for:

- User Login
- Incident Creation
- SLA Configuration
- SLA Monitoring
- Notifications
- Reports & Dashboard
- User Acceptance Testing

These interfaces represent the expected workflow of an SLA Governance solution but are not connected to a live backend or database.
